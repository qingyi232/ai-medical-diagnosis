一、系统概述
================================================================================

  本系统是一套基于"大模型+RAG技术"的智慧医疗方案生成系统。

  核心功能：
  - 输入患者病例信息，系统自动从医疗知识库中检索相关文献
  - 结合通义千问大模型（Qwen-Plus）生成专业的医疗方案
  - 方案包含病情分析、用药建议、治疗计划等内容
  - 支持医生对方案进行审阅、评分和反馈
  - 支持方案导出为 Word 文档

  技术栈：
  - 前端：Vue3 + Element Plus + Vite
  - 后端：Flask + SQLAlchemy + JWT
  - 数据库：MySQL 8.0
  - 大模型：通义千问 Qwen-Plus
  - 向量检索：FAISS + text-embedding-v3
  - RAG技术：检索增强生成

四、管理员功能说明
================================================================================

【4.1 工作台】

  工作台展示系统全局数据概览：
  - 系统用户数：当前系统中的总用户数
  - 患者总数：系统中的患者数量
  - 知识库文档：知识库中的文档数量
  - 生成方案：已生成的医疗方案数量

  快捷操作：用户管理、知识库管理、方案审核、上传文档

  系统信息：显示向量化文档数、已审核方案数、大模型版本、RAG引擎信息

  最近生成的方案：显示最近5条方案记录
  最近病历记录：显示最近5条病历记录

【4.2 用户管理】（管理员专属）

  管理系统中的所有用户账号。

  功能操作：
  - 搜索用户：在搜索框输入用户名或姓名，按回车搜索
  - 角色筛选：通过下拉框筛选管理员或医生
  - 新增用户：
    1. 点击"新增用户"按钮
    2. 填写用户名、密码、姓名、角色、科室等信息
    3. 点击"确定"保存
  - 编辑用户：点击用户行的"编辑"按钮，修改信息后保存
  - 禁用/启用：点击"禁用"按钮可以禁止用户登录，再次点击可恢复
  - 删除用户：点击"删除"按钮，确认后删除（不能删除自己）

【4.3 知识库管理】（管理员完整权限）

  管理医疗知识库文档，这些文档是RAG检索的数据来源。

  文档类型：临床指南、病例文档、教材、论文

  功能操作：
  - 搜索文档：输入关键词搜索文档标题
  - 类型筛选：按文档类型筛选
  - 新增文档：
    1. 点击"新增文档"按钮
    2. 填写标题、类型、疾病分类、描述
    3. 在"文档内容"中粘贴医疗知识文本
    4. 点击"保存"
  - 上传文件：
    1. 点击"上传文件"按钮
    2. 选择文件（支持 txt、pdf、docx、md 格式）
    3. 填写标题、类型等信息
    4. 点击"上传"
  - 向量化：
    ★★★ 非常重要 ★★★
    新增或上传的文档必须点击"向量化"按钮才能被方案生成功能检索到。
    向量化过程：将文档内容切分为片段 → 调用通义千问Embedding API生成向量 → 
    存入FAISS向量库 → 支持语义检索
  - 查看详情：点击"查看"按钮，可查看文档完整内容和切片详情
  - 编辑文档：点击"编辑"按钮，可修改文档信息和内容
  - 删除文档：点击"删除"按钮，确认后删除（会同时删除向量数据）

【4.4 方案管理】

  查看和管理所有生成的医疗方案。

  功能操作：
  - 搜索方案：输入关键词搜索方案标题
  - 状态筛选：按状态筛选（已生成、已审阅、已采纳、已驳回）
  - 查看详情：点击方案标题或"查看"按钮
  - 采纳方案：点击"采纳"按钮，将方案标记为已采纳
  - 删除方案：点击"删除"按钮

五、医生功能说明
================================================================================

【5.1 工作台】

  工作台展示与医生工作相关的数据：
  - 患者总数、病历记录数、知识库文档数、生成方案数

  快捷操作：生成方案、新增患者、录入病历、查看知识库

【5.2 患者管理】（医生专属）

  管理患者基本信息。

  功能操作：
  - 搜索患者：输入姓名或身份证号搜索
  - 新增患者：
    1. 点击"新增患者"按钮
    2. 填写姓名、性别、年龄、联系电话、血型等信息
    3. 填写过敏史、既往病史、家族病史
    4. 点击"确定"保存
  - 查看详情：点击患者姓名，进入患者详情页
    详情页显示患者基本信息和关联的所有病历记录
    可在此页面直接新增病历
  - 编辑患者：点击"编辑"按钮修改患者信息
  - 删除患者：点击"删除"按钮

【5.3 病历管理】（医生专属）

  管理患者的病历记录。

  功能操作：
  - 搜索病历：输入病历编号、患者姓名或诊断搜索
  - 查看详情：点击病历编号进入详情页
    详情页显示完整的病历信息，包括：
    主诉、现病史、既往史、体格检查、实验室检查、影像学检查、诊断
  - 生成方案：在病历详情页点击"基于此病历生成方案"按钮
    系统会自动将病历信息填充到方案生成页面
  - 删除病历：点击"删除"按钮

【5.4 知识库】（医生只读）

  医生可以查看和搜索知识库中的文档，但不能新增、编辑、删除或向量化。
  - 搜索文档
  - 按类型筛选
  - 查看文档详情（内容和切片信息）

【5.5 方案生成】（医生专属，★核心功能★）

  这是系统的核心功能，基于病例信息生成智慧医疗方案。

  使用步骤：
  1. 在左侧"病例信息输入"区域输入方案标题
  2. 在文本框中输入或粘贴完整的病例信息，包括：
     - 主诉（主要症状及持续时间）
     - 现病史（详细病情描述）
     - 既往史（过往疾病、手术史）
     - 体格检查（生命体征及体查结果）
     - 实验室检查（血常规、生化等）
     - 影像学检查（CT、超声等）
     - 当前诊断
  3. 可选操作：点击"知识库检索"按钮，预览知识库中的相关文献
  4. 点击"生成智慧医疗方案"按钮
  5. 等待系统处理（通常需要10-30秒）：
     - 第一步：病例信息解析
     - 第二步：知识库语义检索（RAG）
     - 第三步：大模型方案推理
  6. 生成完成后，右侧显示：
     - 方案内容（Markdown格式渲染）
     - 知识库匹配度
     - 参考来源（来自哪些知识库文档）
     - 免责声明
  7. 可以点击"导出Word"将方案导出为Word文档
  8. 可以点击"查看详情"进入方案详情页

  快捷入口：
  - 从病历详情页点击"基于此病历生成方案"，会自动填充病例信息
  - 从患者详情页选择病历后也可跳转

【5.6 方案管理】

  管理自己生成的所有方案。

  功能操作：
  - 搜索方案：输入关键词搜索
  - 状态筛选：按状态筛选
  - 查看详情：进入方案详情页，可查看完整方案内容
  - 导出Word：点击"导出Word"按钮下载方案文档
  - 医生反馈：
    1. 在详情页左下角"医生反馈"区域
    2. 给方案评分（1-5星）
    3. 填写反馈意见
    4. 点击"采纳方案"、"标记已阅"或"驳回方案"
  - 删除方案：点击"删除"按钮

六、核心功能详解：方案生成
================================================================================

【技术流程】

  用户输入病例 → 文本预处理 → FAISS向量检索匹配知识库 → 
  拼接检索结果作为上下文 → 调用通义千问Qwen-Plus大模型 → 
  生成结构化医疗方案 → 保存到数据库 → 展示给用户

【生成方案的内容结构】

  通义千问大模型会生成以下结构的方案：
  - 病情分析：对患者病例的综合分析
  - 诊断评估：对当前诊断的专业评估
  - 治疗方案：
    - 药物治疗（药名、剂量、用法）
    - 非药物治疗
  - 注意事项与随访计划
  - 方案依据说明

【知识库检索原理】

  系统使用RAG（检索增强生成）技术：
  1. 将知识库文档切分为小段文本（约500字/段）
  2. 使用通义千问text-embedding-v3模型将文本转换为1024维向量
  3. 存储在FAISS向量索引中
  4. 用户输入病例后，同样转换为向量
  5. 在FAISS中进行余弦相似度检索，找到最相关的5个文本片段
  6. 将这些片段作为上下文提供给大模型，提高方案的专业性和准确性
